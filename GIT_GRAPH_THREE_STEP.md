# Git Graph 读图三步法

## 1. 先找分支标签

先看这些标签挂在哪里：

- `main`
- `dev`
- `origin/main`
- `origin/dev`

它们表示：

“这些分支指针当前停在哪个提交上。”

## 2. 再看分叉点

顺着某条分支线往下看：

- 原来和 `main` 走同一条线
- 到某个点开始岔开
- 那个地方就是这条分支“分出来”的位置

一句话：

看到线条第一次岔开，就是“从哪儿分出来”。

## 3. 最后看合并点

重点找这些提交：

- `Merge pull request ... from .../dev`
- `Merge branch 'main' into dev`

它们表示历史在这里发生了合并。

一句话：

看到 `Merge ...` 提交，通常就是“在哪儿合回去”或“在哪儿把别的分支合进来”。

## 最小判断法

- 看标签：知道谁现在在哪
- 看分叉：知道从哪儿分出来
- 看 `Merge ...`：知道在哪儿合回去

## 套到 `dev` 和 `main`

- `dev` 从哪儿分出来：看它第一次脱离 `main` 主线的位置
- `dev` 在哪儿合回 `main`：看 `main` 上的 `Merge pull request ... from .../dev`
